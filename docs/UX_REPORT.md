# 🎮 AI Arena 用户体验报告

**产品体验官：太子**  
**日期：2026-06-26**  
**版本：v0.1.0**  
**测试环境：Windows 10，Electron 桌面应用**

---

## 一、用户旅程分析

### 🚀 Step 1：首次启动（双击 exe）

**用户视角：**  
"我双击了 AI Arena 的图标，看到一个深色界面加载了……大概等了几秒钟，出来了。"

**代码分析：**
- `electron/main.js` 启动时先 spawn Python 后端（uvicorn），等待最多 8 秒（`setTimeout(resolve, 8000)`）
- 然后通过 `waitForBackend()` 轮询 HTTP（最多 40 次 × 500ms = 20 秒）
- **最坏情况用户可能等待 28 秒**，期间没有任何视觉反馈——窗口 `show: false`，完全黑屏
- 没有启动画面（splash screen），没有加载进度条

**问题：**
- ❌ **严重**：启动时黑屏长达 8-28 秒，用户会以为程序卡死了
- ❌ **严重**：没有 splash screen 或加载动画
- ❌ **中等**：如果 Python 没有安装或 uvicorn 未找到，启动失败是静默的——用户看到黑屏然后什么都没有

---

### ⚙️ Step 2：配置 API Key

**用户视角：**  
"打开后默认在'配置'页面。我看到一个空的模型列表，写着'还没有配置任何 AI 模型'，下面有个'+ 添加模型'按钮。我点了按钮，出来一个表单，要填 Base URL、API Key、模型名称、名称+Emoji……"

**代码分析：**
- 默认进入 `config` 页面（`App.currentPage = 'config'`），首次打开列表为空
- 空状态有文字提示"还没有配置任何 AI 模型"和"点击下方按钮添加一个模型"
- 添加后需要填写 4 个字段：Base URL、API Key、模型名称、名称+Emoji

**问题：**
- ❌ **严重**：没有任何新手引导。"Base URL"是什么？"模型名称"填什么？普通大学生根本不知道
- ❌ **严重**：没有预置常见服务商的模板（如 OpenAI、DeepSeek、通义千问等），用户需要自己查文档
- ❌ **中等**：API Key 是 `type="password"` 显示为圆点，但没有"显示/隐藏"切换按钮，用户可能输错了都不知道
- ❌ **中等**：保存是自动的（`change` 事件触发），但没有任何"已保存"的视觉反馈，用户不确定是否保存成功
- ❌ **中等**：没有提供"去哪里获取 API Key"的帮助链接或说明
- ❌ **小问题**：默认 Emoji 是 🤖，但没有任何提示用户可以自定义

---

### 🎯 Step 3：选择场景

**用户视角：**  
"我配好了模型，点了'场景'tab。看到 5 个场景卡片：狼人杀🐺、辩论赛🗣️、知识问答🧠、代码对决💻、故事接龙📖。每个卡片有名字、描述和人数要求。"

**代码分析：**
- 场景卡片展示了 emoji、名称、描述、人数范围
- 点击卡片进入玩家配置界面
- 默认自动分配玩家（从已配置模型中循环取）

**问题：**
- ❌ **中等**：没有"推荐"或"难度"标识，新用户不知道选哪个好。狼人杀要 6 人，如果只配了 2 个模型怎么办？
- ❌ **中等**：如果配置的模型数量少于场景 `min_players`，玩家列表中会出现"未配置"的模型，但没有明确的错误提示
- ❌ **中等**：玩家配置中的"人设"字段（personality）没有任何说明，用户不知道这是干什么的、填什么好
- ❌ **小问题**：没有场景预览或示例回放，用户不知道游戏实际效果
- ❌ **小问题**：场景卡片没有按难度或推荐排序

---

### ⚔️ Step 4：开始游戏

**用户视角：**  
"我选了辩论赛，点了'开始对战'。按钮变成了'⏳ 启动中...'，然后自动跳到了'观战'页面。"

**代码分析：**
- `startGame()` 调用 `API.startGame()` 后切换到 arena 页面
- 后端通过 `asyncio.create_task` 异步启动游戏
- 前端通过 WebSocket 接收实时事件

**问题：**
- ❌ **中等**：启动后没有任何"正在加载"的过渡动画，页面切换有点突兀
- ❌ **中等**：如果 API Key 无效或模型响应超时，错误信息通过 WebSocket 推送为系统事件，但用户可能不知道是 API 的问题
- ❌ **小问题**：`btn-start-game` 在 `finally` 中恢复状态，但如果成功跳转到 arena 页面，按钮状态恢复是无意义的（用户已经不在那个页面了）

---

### 👀 Step 5：观战体验

**用户视角：**  
"观战页面上面有场景名和阶段，中间是消息流，下面是玩家状态栏。AI 们开始说话了，每条消息有头像、名字和内容。"

**代码分析：**
- 消息流通过 WebSocket 实时推送
- 支持 6 种事件类型：system、speech、vote、death、phase_change、game_over
- speech 事件有气泡样式，带玩家 emoji 和颜色
- 有自动滚动到底部

**问题：**
- ❌ **中等**：`arena-players` 区域（玩家状态栏）在 HTML 中有占位，但 `Arena.init()` 中没有渲染玩家卡片的逻辑——`onGameStart()` 也没有填充玩家状态。**底部玩家栏是空的**
- ❌ **中等**：没有打字机效果或消息逐字显示，AI 的长文本瞬间出现，缺少"正在思考"的感觉
- ❌ **中等**：暂停按钮（`btn-pause`）绑定了 `disabled` 属性但**没有任何点击事件处理**——点了没反应
- ❌ **小问题**：AI 发言没有时间戳，用户不知道每条消息的间隔
- ❌ **小问题**：没有"正在输入"的动画指示器（AI 思考时用户看不到任何反馈）
- ❌ **小问题**：长消息没有折叠机制，辩论赛中 AI 可能输出很长的论述，占据大量空间

---

### 🏁 Step 6：游戏结束

**用户视角：**  
"游戏结束了，显示了一个大的'🎉'和获胜信息。然后呢？我该干什么？"

**代码分析：**
- `addGameOverEvent()` 显示获胜信息，禁用停止按钮，更新阶段为"游戏结束"
- **没有**"再来一局"按钮、"返回场景选择"按钮、"导出结果"功能

**问题：**
- ❌ **严重**：游戏结束后没有任何引导操作——没有"再来一局"、"换个场景"、"分享结果"按钮。用户完全不知道下一步该干什么
- ❌ **中等**：游戏结束后观战页面的消息流保留了，但如果用户切换到其他 tab 再回来，消息还在但状态可能不一致
- ❌ **中等**：没有游戏结果的结构化展示（如投票统计、角色揭示、关键回合回顾）

## 三、改进建议

### 🚀 P0 — 立即修复（影响核心可用性）

#### 1. 添加启动画面（Splash Screen）
```
在 BrowserWindow 创建时 show: true，加载一个本地 HTML splash：
- 显示 AI Arena logo + 加载动画
- 文字提示"正在启动后端服务..."
- 如果超过 15 秒未就绪，显示错误提示和解决建议
```

#### 2. 添加首次使用引导
```
首次打开时（检测 config.models 为空）：
- 显示一个欢迎 modal/overlay
- 分 3 步引导：① 选择服务商 → ② 填入 API Key → ③ 测试连接
- 提供常见服务商的预设模板（DeepSeek、OpenAI、通义千问等）
```

#### 3. 游戏结束后的引导
```
game_over 事件后：
- 显示"再来一局"按钮（重开同一场景）
- 显示"换个场景"按钮（返回场景选择）
- 显示游戏结果摘要（谁赢了、关键数据）
```

#### 4. 填充玩家状态栏
```
onGameStart() 中：
- 根据 players 数据渲染玩家状态卡片
- 显示 emoji + 名字 + 存活状态
- 游戏过程中实时更新存活/淘汰状态
```

### 🔧 P1 — 尽快修复（影响体验流畅度）

#### 5. 修复暂停按钮
- 要么实现暂停功能（调用后端 pause API），要么移除按钮避免误导

#### 6. 添加配置保存反馈
- 保存成功后显示一个轻量 toast："✅ 已保存"
- 或在输入框右侧显示一个短暂的 ✓ 动画

#### 7. API Key 显示/隐藏切换
- 在 API Key 输入框右侧添加一个 👁️ 按钮，切换 `type="text"` / `type="password"`

#### 8. 添加"正在思考"指示
- 在 AI 响应前显示 "🤔 {player_name} 正在思考..." 的动画
- 可以用一个带脉冲动画的占位气泡

#### 9. 改善错误提示
- 用应用内 toast/notification 替代 `alert()`
- API Key 无效时给出具体提示："请检查 API Key 是否正确"
- Python 未安装时显示安装指引

#### 10. WebSocket 连接状态指示
- 在导航栏或状态栏显示连接状态（🟢 已连接 / 🔴 断开）
- 断连时显示"正在重连..."提示

### ✨ P2 — 优化打磨（提升品质感）

#### 12. 消息添加时间戳
- 在每条 speech 事件旁显示相对时间（如 "刚刚"、"3秒前"）

#### 13. 长消息折叠
- 超过 3 行的消息默认折叠，显示"展开更多 ▼"

## 六、总结

AI Arena 的核心概念非常有吸引力——让不同的 AI 模型互相 PK，这在市面上是独特的。代码架构也相当清晰（前后端分离、WebSocket 实时通信、场景插件化设计），为后续扩展打下了好基础。

但目前产品处于"能跑但不好用"的阶段。最大的问题是**新用户引导完全缺失**——一个不熟悉 API 的大学生打开应用后，大概率会在配置页面卡住，然后关闭应用。

**最关键的三件事：**
1. 🔥 添加启动画面，消除黑屏焦虑
2. 🔥 添加模型预设 + 新手引导，降低使用门槛
3. 🔥 游戏结束后添加操作引导，形成完整闭环

做好这三件事，产品的可用性会有质的飞跃。
