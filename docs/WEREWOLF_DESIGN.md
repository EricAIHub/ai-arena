## 1. 概述

狼人杀 v2 在 v1 基础上进行三大增强：

| 维度 | v1 | v2 |
|------|-----|-----|
| 角色 | 4 种（狼人/预言家/医生/平民） | 7 种（+猎人/女巫/守卫） |
| 用户参与 | 纯 AI 自动对战 | 支持用户作为玩家参与 |
| AI 策略 | 基础 prompt 驱动 | 角色差异化策略 + 记忆分析 |

### 核心设计理念

- **上帝视角**：用户默认以观战者身份观看 AI 之间的博弈
- **沉浸参与**：用户可选择一个角色加入游戏，与 AI 同台竞技
- **无缝切换**：用户可随时在观战和参与模式之间切换
- **AI 智能**：每个 AI 都有基于角色的差异化策略和记忆能力

## 2. 游戏模式

### 2.1 纯观战模式（Spectator）

- 用户不占用玩家席位
- 所有玩家均为 AI
- 用户可以看到所有角色的私密信息（上帝视角）
- 可以加速/暂停游戏

### 2.2 参与模式（Player）

- 用户选择一个玩家席位加入
- 该席位标记为 `is_human = True`
- 轮到用户行动时弹出输入界面
- 其他时候 AI 自动行动
- 用户可以随时切换回观战模式（AI 接管）

### 2.3 模式切换

```
观战模式 → 用户点击"加入游戏" → 选择席位 → 参与模式
参与模式 → 用户点击"切换观战" → AI 接管该席位 → 观战模式
```

当用户从参与切换到观战时，AI 接管的策略：
- 复用该席位已有的 model_name 配置
- 继承之前的发言历史和角色信息
- AI 接管后的首次发言会说明"暂时由 AI 代理"

### 3.2 角色详细说明

#### 🐺 狼人（Werewolf）

- **阵营**：狼人阵营
- **技能**：每晚与同伴协商选择一名好人击杀
- **胜利条件**：存活狼人数量 ≥ 存活好人数量
- **策略要点**：
  - 白天伪装成好人
  - 适当"怀疑"一个非狼同伴来建立信任
  - 投票时不投给同伴
  - 优先在晚上击杀跳出来的预言家
  - 可以假装是医生或平民

#### 🔮 预言家（Seer）

- **阵营**：好人阵营
- **技能**：每晚查验一名玩家的身份（狼人/好人）
- **策略要点**：
  - 优先查验发言最可疑的玩家
  - 查验到狼人后权衡是否立即公开
  - 可以用隐晦方式暗示查验结果
  - 留意医生的保护目标

#### 💊 医生（Doctor）

- **阵营**：好人阵营
- **技能**：每晚保护一名玩家免受狼人杀害，不能连续两晚保护同一人
- **策略要点**：
  - 优先保护高价值目标（预言家等）
  - 轮换保护对象
  - 适当暗示身份但不暴露
  - 观察狼人投票模式推断目标

#### 🎯 猎人（Hunter）

- **阵营**：好人阵营
- **技能**：被淘汰（投票或狼人击杀）时，可以选择带走一名玩家
- **触发条件**：
  - 被投票淘汰时：立即触发
  - 被狼人击杀时：天亮后触发
  - 被女巫毒杀时：**不触发**（毒药压制猎人技能）
- **策略要点**：
  - 淘汰时要冷静分析，带走最可疑的人
  - 如果是狼人猎人（扩展规则），可以带走好人

#### 🧪 女巫（Witch）

- **阵营**：好人阵营
- **技能**：
  - **解药**：可以救活被狼人击杀的玩家（全局仅 1 瓶）
  - **毒药**：可以毒杀一名玩家（全局仅 1 瓶）
  - 每晚只能使用一种药
- **限制**：
  - 解药和毒药各只有 1 瓶
  - 使用后永久消耗
- **策略要点**：
  - 解药优先救高价值目标
  - 毒药要谨慎使用，确认目标是狼人
  - 可以在白天暗示身份引导局势

#### 🛡️ 守卫（Guard）

- **阵营**：好人阵营
- **技能**：每晚可以守护一名玩家，使其免受狼人击杀
- **限制**：
  - 不能连续两晚守护同一人
  - 守护和医生保护可以叠加（双重保护）
- **策略要点**：
  - 与医生错开保护对象
  - 优先守护高价值目标
  - 不要暴露身份

#### 👤 平民（Villager）

- **阵营**：好人阵营
- **技能**：无特殊技能
- **策略要点**：
  - 分析发言逻辑找出矛盾
  - 注意投票一致性
  - 保护预言家和医生的身份信息
  - 被怀疑时冷静辩护

### 3.3 角色优先级（夜晚行动顺序）

```
1. 守卫选择守护对象
2. 狼人选择击杀目标（协商）
3. 女巫选择是否用药（救人/毒人）
4. 预言家查验
```

### 3.4 角色优先级（白天行动顺序）

```
1. 公布夜晚结果
2. 所有存活玩家轮流发言（讨论）
3. 投票淘汰
4. 被淘汰者发表遗言
5. 如果被淘汰者是猎人，触发猎人技能
```

## 4. 游戏流程

### 4.1 完整流程图

```
游戏开始
    │
    ▼
┌─────────────────────────────────────┐
│           夜晚阶段 (NIGHT)           │
│                                     │
│  1. 守卫选择守护对象                  │
│  2. 狼人协商选择击杀目标              │
│  3. 女巫选择是否使用解药/毒药         │
│  4. 预言家查验一名玩家               │
│                                     │
│  结算：                              │
│  - 守卫 + 医生双重保护 → 平安夜      │
│  - 女巫救人 → 平安夜                 │
│  - 无保护 → 被击杀者死亡             │
│  - 女巫毒药 → 目标死亡               │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│         白天讨论阶段 (DISCUSSION)     │
│                                     │
│  1. 公布夜晚结果（谁死了/平安夜）     │
│  2. 所有存活玩家轮流发言              │
│  3. 被击杀者发表遗言（如果有）        │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│          投票阶段 (VOTE)             │
│                                     │
│  1. 所有存活玩家投票                  │
│  2. 统计票数，最高票者被淘汰          │
│  3. 平票则无人被淘汰                  │
│  4. 被淘汰者发表遗言                  │
│  5. 如果被淘汰者是猎人，触发技能      │
└─────────────────────────────────────┘
    │
    ▼
  检查胜负
    │
    ├── 狼人全部死亡 → 好人获胜
    ├── 狼人数量 ≥ 好人数量 → 狼人获胜
    └── 继续 → 回到夜晚阶段
```

### 4.2 夜晚结算逻辑

```
狼人击杀目标 T
    │
    ├── 守卫守护了 T？ ──是──→ T 幸存
    │
    ├── 医生保护了 T？ ──是──→ T 幸存
    │
    └── 都没有 ──→ T 死亡
                    │
                    └── 女巫使用解药救 T？ ──是──→ T 幸存
                                               └──否──→ T 死亡

女巫使用毒药目标 P ──→ P 死亡（无法被守卫/医生保护）
```

### 4.3 猎人技能触发

```
猎人被淘汰
    │
    ├── 被女巫毒杀？ ──是──→ 不触发技能
    │
    └── 其他原因 ──→ 触发技能
                     │
                     ├── 用户玩家 → 弹出选择界面
                     └── AI 玩家 → AI 自动选择
                     │
                     └── 带走目标玩家
```

## 5. 用户参与机制

### 5.1 加入游戏

```json
// POST /api/game/start
{
  "scenario": "werewolf_v2",
  "players": [
    {
      "id": "human_1",
      "name": "玩家1",
      "model_name": "",
      "is_human": true,       // 标记为人类玩家
      "emoji": "😎",
      "color": "#FF6B6B"
    },
    {
      "id": "ai_1",
      "name": "AI-Alpha",
      "model_name": "gpt-4",
      "is_human": false,
      "emoji": "🤖",
      "color": "#4ECDC4"
    }
    // ... 更多 AI 玩家
  ]
}
```

### 5.2 用户输入流程

```
游戏引擎执行到用户玩家的回合
    │
    ▼
发送 waiting_input 事件（通过 WebSocket）
{
  "type": "waiting_input",
  "player_id": "human_1",
  "content": "轮到你发言了",
  "input_type": "speech",       // speech | vote | witch_save | witch_poison | hunter_shoot | guard_protect | seer_check
  "candidates": [...]           // 可选列表（投票/选择时）
}
    │
    ▼
前端显示输入界面
    │
    ▼
用户提交输入
POST /api/game/input
{
  "text": "我觉得3号很可疑"    // 发言内容
  // 或
  "choice": "player_3"         // 选择目标
}
    │
    ▼
游戏引擎恢复执行
```

### 5.3 输入等待机制

游戏引擎通过 `asyncio.Future` 实现输入等待：

```python
# 等待用户输入（不阻塞事件循环）
result = await engine.wait_for_user_input(
    player_id="human_1",
    prompt="轮到你发言了",
    input_type="speech",
    timeout=120  # 2分钟超时
)

# 用户通过 API 提交
await engine.submit_user_input(text="我觉得3号很可疑")
```

### 5.4 超时处理

- 发言超时：120 秒后自动跳过（AI 接管发言）
- 投票超时：60 秒后自动弃票
- 技能使用超时：30 秒后默认不使用

### 5.5 模式切换

```json
// POST /api/game/mode
{
  "mode": "spectator"     // spectator | player
}
```

切换到观战模式时：
- AI 接管用户席位
- 发送系统消息通知
- 用户可以看到所有角色信息（上帝视角）

## 6. AI 策略体系

### 6.1 策略架构

每个 AI 玩家的决策基于：
1. **角色信息**：自己的角色和技能
2. **历史记忆**：之前的发言、投票、死亡事件
3. **身份推理**：基于已知信息推断其他玩家身份
4. **角色策略**：针对不同角色的差异化行为

### 6.2 狼人策略

#### 伪装策略
- 白天发言模仿好人的推理模式
- 不主动暴露同伴信息
- 适当"怀疑"一个非狼同伴来建立信任

#### 协作策略
- 夜晚与同伴协商击杀目标
- 投票时跟大多数人的节奏，不投给同伴
- 如果有预言家跳出来，优先在晚上击杀

#### 反踩策略
- 怀疑揭发狼人的好人
- 质疑预言家查验结果的真实性
- 制造混乱和分裂

### 6.3 好人策略

#### 预言家策略
- 优先查验发言最可疑的玩家
- 权衡是否立即公开查验结果
- 可以用隐晦方式暗示查验结果
- 如果被狼人围攻，考虑公开身份

#### 医生策略
- 优先保护高价值目标（预言家等）
- 轮换保护对象
- 不要暴露身份

#### 守卫策略
- 与医生错开保护对象
- 优先守护高价值目标
- 不要暴露身份

#### 女巫策略
- 解药优先救高价值目标
- 毒药要谨慎使用
- 可以在白天暗示身份引导局势

#### 平民策略
- 分析发言逻辑找出矛盾
- 注意投票一致性
- 保护预言家和医生的身份信息

### 6.4 通用记忆分析

AI 会记住并分析：
- **发言历史**：每个玩家在每轮的发言内容
- **投票记录**：谁投了谁，投票模式
- **死亡事件**：谁在什么情况下死亡
- **身份线索**：预言家的查验结果、女巫的用药情况等
- **矛盾检测**：检测发言中的逻辑矛盾

## 7. 胜负判定

### 7.1 狼人获胜条件

```
存活狼人数量 >= 存活好人数量
```

### 7.2 好人获胜条件

```
所有狼人都被淘汰
```

### 7.3 特殊情况

- 如果猎人被淘汰时带走了最后一个狼人，好人获胜
- 如果女巫毒死了最后一个狼人，好人获胜
- 如果投票淘汰了最后一个狼人，好人获胜

## 8. 事件系统

### 8.1 事件类型

| 类型 | 说明 | 数据字段 |
|------|------|---------|
| `system` | 系统消息 | `content` |
| `phase_change` | 阶段切换 | `phase` |
| `speech` | 玩家发言 | `player_id`, `content` |
| `vote` | 投票 | `player_id`, `target_id` |
| `death` | 死亡 | `player_id`, `role` |
| `thinking` | AI 思考中 | `player_id` |
| `waiting_input` | 等待用户输入 | `player_id`, `input_type`, `candidates` |
| `skill_use` | 技能使用 | `player_id`, `skill`, `target_id` |
| `game_over` | 游戏结束 | `winner` |
| `error` | 错误 | `content`, `code` |

### 8.2 事件推送

所有事件通过 WebSocket 实时推送到前端：

```json
{
  "type": "speech",
  "player_id": "ai_1",
  "player_name": "AI-Alpha",
  "player_emoji": "🤖",
  "player_color": "#4ECDC4",
  "content": "我觉得3号玩家很可疑，他的发言前后矛盾。",
  "phase": "day_discussion"
}
```

## 10. 前端交互规范

### 10.1 观战模式界面

- 游戏区域：显示所有玩家头像和状态
- 事件流：实时滚动的游戏事件日志
- 角色面板：显示每个玩家的角色（上帝视角）
- 控制栏：开始/停止/重置/加速按钮

### 10.2 参与模式界面

- 轮到用户时：弹出输入框（发言）或选择界面（投票/技能）
- 其他时候：与观战模式相同，但隐藏其他玩家的角色信息
- 状态提示：显示当前轮到谁、用户是什么角色

### 10.3 输入界面

#### 发言输入
- 文本输入框
- 发送按钮
- 倒计时（120 秒）

#### 投票选择
- 玩家列表（可点击选择）
- 确认按钮
- 倒计时（60 秒）

#### 技能使用
- 根据技能类型显示不同界面
- 猎人：选择带走的玩家
- 女巫：选择是否使用解药/毒药，选择目标
- 守卫：选择守护的玩家
- 预言家：选择查验的玩家
- 医生：选择保护的玩家
