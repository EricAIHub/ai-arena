## 1. 功能分级清单

### 1.1 极客版功能（核心）

> 原则：**能跑就行，好看是次要的**。极客版是验证产品假设的最小可行产品。

| 功能 | 当前状态 | 说明 |
|------|---------|------|
| 🐺 狼人杀完整流程 | ✅ 已实现 | 角色分配→夜晚→讨论→投票→胜负判定 |
| 🎤 辩论赛 | ✅ 已实现 | 正反方对抗，多轮讨论 |
| 💻 代码对决 | ✅ 已实现 | 同题竞赛 |
| 🧠 知识问答 | ✅ 已实现 | 抢答计分 |
| 📖 故事接龙 | ✅ 已实现 | 接力创作 |
| 🤖 AI 自动对战 | ✅ 已实现 | 所有玩家均为 AI，自动执行 |
| 👁️ 实时观战（消息流） | ✅ 已实现 | WebSocket 推送，气泡样式消息流 |
| ⚙️ 配置管理 | ✅ 已实现 | API Key / 模型配置 |
| 🎨 主题切换 | ✅ 已实现 | 纯净 / 豪华 / 柔和 三套主题 |
| 🔌 WebSocket 实时通信 | ✅ 已实现 | 事件广播 + 心跳保活 |
| 📊 游戏状态快照 | ✅ 已实现 | 保存/恢复游戏状态 |
| 🎭 自定义人设 | ✅ 已实现 | 每个玩家可设置性格描述 |

### 1.2 增强功能（极客版验证 → 高级版升级）

> 原则：先在极客版用最简方式验证，确认用户喜欢后再在高级版加动效包装。

| 功能 | 极客版形态 | 高级版升级 | 优先级 |
|------|-----------|-----------|--------|
| 🙋 用户参与机制 | 文字输入框，用户手动发言 | 智能提示 + 语音输入 + 表情包 | P1 |
| 🗡️ 更多角色（猎人/女巫/守卫） | YAML 配置 + 简单文字交互 | 角色专属动画 + 技能特效 | P1 |
| 🧠 AI 策略增强 | 优化 prompt，加入记忆/推理链 | AI 人格进化系统（随对局成长） | P1 |
| ⏪ 游戏回放 | JSON 事件流导出 | 可视化时间轴 + 倍速播放 + 标记关键时刻 | P2 |
| 📈 模型战绩统计 | 本地 JSON 记录胜负场次 | 在线排行榜 + ELO 评分 + 数据可视化面板 | P2 |
| 🎯 智能推荐 | — | 根据用户历史推荐场景和模型组合 | P3 |
| 🌐 多语言支持 | — | 中/英/日三语切换 | P3 |

### 1.3 娱乐功能（高级版专属）

> 原则：这些功能不影响核心玩法，但能大幅提升"看"的体验和分享欲。

| 功能 | 说明 | 实现难度 |

| 🎙️ 字幕旁白 | AI 实时生成解说词，TTS 朗读 | 中等 |
| 🎭 角色动作 | 捂眼睛、指人、思考、惊讶等动画 | 中等 |
| ✨ 粒子背景 | 根据游戏阶段变化的动态粒子效果 | 简单 |
| 🃏 3D 卡片翻转 | 角色卡/投票卡的 3D 翻转动画 | 中等 |
| ⌨️ 打字机效果 | AI 发言逐字显示，模拟打字感 | 简单 |
| 💥 爆炸/烟花动效 | 淘汰时爆炸、胜利时烟花 | 简单 |
| 🧑‍🎤 角色模型库 | 可爱的像素/卡通角色形象 | 困难 |
| 🔊 音效系统 | 背景音乐 + 阶段音效 + 淘汰音效 | 简单 |
| 🎬 赛事回放剧场 | 带旁白、音乐、特效的完整回放 | 困难 |
| 🌙 日夜交替场景 | 背景随阶段变化（白天/夜晚/黄昏） | 中等 |
| 📱 竖屏模式 | 适配手机竖屏的观战体验 | 中等 |

## 2. 创意功能列表

### 2.1 高价值功能（提升核心体验）

#### 💡 创意 1：AI 人格进化系统

| 项目 | 内容 |
|------|------|
| **描述** | 每个 AI 玩家拥有人格档案，随对局积累经验。狼人局打多了会变得更狡猾，预言家打多了会更谨慎。人格数据持久化到本地 JSON。 |
| **用户价值** | 让 AI 不再是"每局重新开始的陌生人"，用户会想看"我养的 DeepSeek 成长到什么水平了" |
| **实现难度** | 中等 |
| **建议版本** | 极客版（基础记忆）→ 高级版（可视化人格树） |

#### 💡 创意 2：实时 AI 思维链展示

| 项目 | 内容 |
|------|------|
| **描述** | 在观战面板中，可折叠查看每个 AI 的"内心独白"——即 AI 的推理过程（chain-of-thought）。比如狼人在决定杀谁时，会展示"张三上轮发言太积极，可能是预言家，优先击杀"。 |
| **用户价值** | 这是 AI Arena 最大的差异化卖点——用户不仅看到结果，还能看到 AI 的思考过程。对内容创作者来说是绝佳素材。 |
| **实现难度** | 中等（需要从 AI 回复中提取推理部分） |
| **建议版本** | 极客版（纯文本展示）→ 高级版（气泡动画 + 高亮关键词） |

#### 💡 创意 3：模型盲测模式

| 项目 | 内容 |
|------|------|
| **描述** | 隐藏所有模型名称，用户只看到"选手A/B/C"。游戏结束后让用户猜测哪个是 GPT、哪个是 DeepSeek。猜对有成就徽章。 |
| **用户价值** | 消除品牌偏见，让用户真正基于表现评判 AI 能力。社交传播性极强——"你猜哪个是 ChatGPT？" |
| **实现难度** | 简单 |
| **建议版本** | 极客版 |

#### 💡 创意 4：自定义场景编辑器

| 项目 | 内容 |
|------|------|
| **描述** | 用户通过表单或 YAML 编辑器创建自定义场景：定义角色、阶段流程、胜利条件、AI prompt 模板。支持导入/导出场景配置。 |
| **用户价值** | 从"用别人的场景"到"创造自己的场景"，解锁无限玩法。比如"AI 法庭辩论"、"AI 模拟面试"。 |
| **实现难度** | 困难 |
| **建议版本** | 极客版（YAML 文件）→ 高级版（可视化编辑器） |

#### 💡 创意 5：智能裁判解说

| 项目 | 内容 |
|------|------|
| **描述** | 引入一个"裁判 AI"角色，实时解说比赛。"注意！3号选手的发言和上一轮矛盾了！"裁判风格可选：专业解说 / 毒舌吐槽 / 中二热血。 |
| **用户价值** | 大幅提升观战趣味性，让不懂规则的用户也能看懂比赛。 |
| **实现难度** | 中等 |
| **建议版本** | 高级版（配合 TTS 旁白） |

#### 💡 创意 6：多模型混合组队

| 项目 | 内容 |
|------|------|
| **描述** | 允许同一阵营的 AI 使用不同模型。比如狼人阵营：GPT 负责伪装，DeepSeek 负责推理击杀目标。探索不同模型的"配合"效果。 |
| **用户价值** | 揭示不同模型的特长——谁更适合当狼人（伪装），谁更适合当预言家（推理）。 |
| **实现难度** | 简单（当前架构已支持，只需前端允许混选） |
| **建议版本** | 极客版 |

## 3. 狼人杀玩法增强建议

### 3.1 用户作为"上帝+玩家"的交互设计

> 核心理念：用户既是观察者，也是参与者。不是简单的"看 AI 自己玩"，而是"我来掌控这场 AI 大戏"。

#### 🎮 模式 A：上帝模式（当前模式增强）

| 增强项 | 说明 |
|--------|------|
| **手动触发阶段** | 添加"下一步"按钮，用户决定何时进入夜晚/白天/投票，而不是全自动。紧急时可加速/跳过。 |
| **手动干预事件** | 用户可以"上帝之手"——复活一个玩家、强制淘汰某人、改变投票结果。用于制造戏剧性。 |
| **自定义规则** | 开局前可调整：狼人数量、是否启用猎人/女巫、投票规则（匿名/公开）、是否允许遗言。 |
| **实时弹幕** | 用户在观战时可以发弹幕（本地存储），记录自己的推理和吐槽。 |

#### 🎮 模式 B：用户当玩家

| 设计 | 说明 |
|------|------|
| **用户选角色** | 开局前用户选择想扮演的角色（预言家/狼人/医生等），其余角色由 AI 填充。 |
| **夜晚交互** | 如果用户是狼人，弹出选择面板让用户选择击杀目标。如果用户是预言家，选择查验目标。 |
| **白天发言** | 用户通过文字输入框发言。系统提供"AI 辅助建议"——基于当前局势生成 3 条建议发言，用户可选择或修改。 |
| **投票交互** | 投票阶段展示所有存活玩家的卡片，用户点击投票。 |

#### 🎮 模式 C：用户当裁判

| 设计 | 说明 |
|------|------|
| **自定义判罚** | 用户可以判定"某人发言违规"（如暴露身份），给予警告或直接淘汰。 |
| **添加突发事件** | 裁判可以触发"突发事件"——"天降流星，随机淘汰一人"、"女巫获得双倍解药"。 |
| **赛后点评** | 游戏结束后，用户可以给每个 AI 的表现打分（1-5星），这些评分会影响 AI 的"人格档案"。 |

### 3.3 AI 行为更像真人的策略

> 当前问题：AI 发言太"完美"、太"理性"，缺乏真人玩家的"情绪波动"和"非理性行为"。

#### 🧠 策略 1：情绪系统

| 设计 | 说明 |
|------|------|
| **情绪状态** | 每个 AI 有情绪值：冷静/紧张/愤怒/恐惧/兴奋。情绪会影响发言风格。 |
| **触发条件** | 被怀疑时紧张↑，同伴被淘汰时愤怒↑，成功骗过所有人时兴奋↑。 |
| **行为影响** | 紧张时发言会结巴/重复，愤怒时会攻击性更强，恐惧时会急于辩解。 |
| **实现** | 在 prompt 中加入当前情绪状态，要求 AI 用对应语气发言。 |

#### 🧠 策略 2：记忆与遗忘

| 设计 | 说明 |
|------|------|
| **短期记忆** | 记住最近 2-3 轮的发言和投票，用于推理。 |
| **长期记忆** | 记住关键事件（如"第 1 轮张三说相信李四"），后续可以用来"翻旧账"。 |
| **遗忘机制** | 早期细节会逐渐模糊，AI 可能会"记错"某人的发言，制造戏剧性。 |
| **实现** | 维护一个结构化的记忆摘要，每轮更新，传入 prompt。 |

#### 🧠 策略 3：社交博弈

| 设计 | 说明 |
|------|------|
| **信任网络** | 每个 AI 对其他玩家有信任度（0-100），基于发言和行为动态调整。 |
| **联盟形成** | 好人之间可以形成"临时联盟"，狼人之间可以"卖队友"来获取信任。 |
| **反水机制** | 狼人可以故意暴露一个同伴来获取好人信任（"我验了他是狼人"），增加博弈深度。 |
| **投票策略** | 投票时不只考虑"谁是狼人"，还要考虑"投谁对我最有利"。 |

#### 🧠 策略 4：个性化行为模式

| 设计 | 说明 |
|------|------|
| **玩家类型** | 每个 AI 有一个行为类型：激进型（喜欢跳身份）、保守型（默默分析）、搅浑水型（故意制造混乱）、话痨型（发言超长）。 |
| **一致性** | 同一个模型在不同局中保持相对一致的风格，但会根据局势微调。 |
| **学习能力** | 如果某个策略导致早期淘汰，下次会调整。比如"上次太激进被投票淘汰，这次收敛一点"。 |

---

### 3.4 游戏结束后的回放/复盘功能

#### ⏪ 基础回放（极客版）

| 功能 | 说明 |
|------|------|
| **事件时间轴** | 线性展示所有游戏事件，可点击跳转到任意节点。 |
| **角色身份揭示** | 游戏结束后揭示所有角色身份，在回放中用不同颜色标注狼人/好人。 |
| **关键节点标记** | 自动标记关键事件：首次被怀疑、投票转折点、致命失误。 |
| **JSON 导出** | 完整的游戏事件流可导出为 JSON，方便开发者分析。 |

#### 🎬 高级回放（高级版）

| 功能 | 说明 |
|------|------|
| **可视化时间轴** | 横向时间轴 + 纵向玩家列表的矩阵视图，一目了然。 |
| **倍速播放** | 支持 0.5x / 1x / 2x / 4x 倍速，关键节点自动减速。 |
| **AI 复盘分析** | 游戏结束后，裁判 AI 生成复盘报告："第 2 轮，3 号玩家的发言暴露了狼人身份，但好人阵营没有抓住这个线索..."。 |
| **投票热力图** | 用热力图展示每轮投票的分布，揭示投票趋势。 |
| **发言情绪曲线** | 用折线图展示每个玩家的发言情绪变化，标注情绪拐点。 |
| **"如果当时..."** | 用户可以修改某个节点的决策（如"如果当时投了 3 号"），系统模拟后续发展。 |
| **分享复盘卡** | 生成精美的复盘卡片，包含关键数据和精彩发言，可截图分享。 |
